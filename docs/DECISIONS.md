# Decisions

## 2026-03-16 — Use a recoverability-first repo operating system
Status: accepted

Why:
The project is expanding quickly across multiple domains and needs a built-in method for tracking changes, decisions, and safe restore points.

## 2026-03-16 — Treat Workforce as a platform core first
Status: accepted

Why:
The same backend primitives should support hospitality, housekeeping, maintenance, field service, and other service operations.

## 2026-03-16 — Use widget-first workspaces for the UI
Status: accepted

Why:
Rigid page-first design makes cross-domain customization harder and duplicates UI patterns.
